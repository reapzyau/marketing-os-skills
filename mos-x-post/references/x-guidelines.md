# X (Twitter) Guidelines

Platform-specific formatting, thread structure, and best practices for X/Twitter posts.

---

## Character Limits

- **Single tweet:** 280 characters (free), 25,000 characters (Premium)
- **Optimal single tweet:** Under 200 characters (more QT-able, more readable)
- **Thread length sweet spot:** 7-8 tweets (enough value, not exhausting)
- **Thread range:** 5-12 tweets. Under 5 feels thin. Over 12 loses readers.

---

## Formatting Rules

1. **Brevity wins.** Every word must earn its place. Cut ruthlessly.
2. **White space matters.** Short lines, visual breathing room, lists with bullets.
3. **No links in tweets** (suppresses reach). Put links in replies or use Linktree in bio.
4. **No hashtags.** X algorithm doesn't reward them. They look spammy.
5. **Line breaks between ideas.** One thought per line when possible.
6. **Numbers > words.** "5 things" not "a few things." Specificity wins.

---

## Single Tweet Structure

```
[Hook/statement — punchy, under 200 chars ideal]

[Supporting point or list — if space allows]

[Punchline or reframe — the payoff]
```

**Best single tweet formats:**
- One bold statement + one line of evidence
- 3-item list with a takeaway
- Question that forces internal answer
- Contrarian take in one sentence
- "Old way vs new way" compressed

## Thread Structure

### Anatomy of a High-Performing Thread

```
Tweet 1 (HOOK — 99% of thread success):
[Number] [dead simple] steps to [desired outcome]

[Optional: "even if [objection]"]

A thread:

Tweet 2 (PROMISE):
By the end of this thread, you'll [know/have/understand]:

- [Outcome 1]
- [Outcome 2]
- [Outcome 3]

Let's go:

Tweets 3-N (VALUE — one idea per tweet):
[Step number]. [Header]

[2-3 lines of context/explanation]

[Optional: example or specific detail]

Final Tweet (CTA):
That's a wrap.

TL;DR:
- [Key point 1]
- [Key point 2]
- [Key point 3]

Follow @[handle] for more on [topic].

[Optional: RT tweet 1 to share with your audience]
```

### Thread Rules

1. **Hook tweet determines everything.** Same thread with different hooks = 14X performance difference (Welsh).
2. **Write all headers first.** Get the skeleton right before adding body text.
3. **One idea per tweet.** Don't cram multiple concepts into one tweet.
4. **Each tweet should stand alone.** Readers might see tweet 4 in isolation — it should still make sense.
5. **Don't post anything else on thread day.** Let the thread ride and grow.
6. **Number your steps.** "1." "2." "3." — readers need to know where they are.

---

## Posting Strategy

- **Frequency:** 3-5 tweets per day (singles). 2-4 threads per week.
- **Thread days:** Mid-week and weekends perform well (Welsh used Wed/Thu/Sat/Sun).
- **No posts after a thread.** Let it breathe for the rest of the day.
- **Auto-retweet:** Set milestone (75-100 likes) to auto-RT your own tweet 9 hours later. Un-retweet 24 hours after.
- **Auto-plug:** Set higher milestone (500-1500 likes) to auto-reply with newsletter/product link.

---

## Algorithm Signals

**What X rewards:**
1. **Replies and conversations** — Tweets that start discussions get boosted
2. **Quote tweets** — Shorter tweets get more QTs (under 200 chars)
3. **Bookmarks** — The save equivalent. Valuable content gets bookmarked.
4. **Time on tweet** — Longer dwell time = more distribution
5. **Profile clicks** — If your tweet makes people click your profile, it gets boosted

**What X suppresses:**
- External links (put in replies)
- Hashtags (irrelevant to the algorithm)
- Engagement bait without substance
- Posting identical content repeatedly

---

## Content That Performs on X

**High performers:**
- One-liner hot takes (short, quotable)
- Compressed lists (3-5 items, no fluff)
- Thread tutorials with clear steps
- Contrarian opinions with evidence
- "Old way vs new way" format
- Personal stories in 2-3 sentences

**Low performers:**
- Long single tweets (use a thread instead)
- Link-heavy tweets
- Generic advice without specificity
- Threads without a strong hook
- Motivational platitudes

---

## X-Specific Adaptation Rules

When adapting from long-form content:
- **Compress ruthlessly.** Cut every unnecessary word. "In order to" → "to". "A lot of" → "many".
- **Lead with the punchline.** X rewards the statement, not the buildup.
- **Use platform language.** "Tweet" not "post." Don't say "LinkedIn" on X.
- **Lose the formality.** More casual, more direct, more personality.
- **Any step-by-step content is a thread.** Convert directly.

## Thread-to-Content Waterfall

A single high-performing thread becomes:
```
Thread
  → Carousel (each tweet = 1 slide)
  → Newsletter topic (8 weeks later)
  → YouTube video script
  → Individual tweets (each tweet repurposed as standalone)
```

This is Matt Gray's Thread System — the thread is the atomic content unit. Everything else derives from it.
